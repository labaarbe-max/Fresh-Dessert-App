## Règles communes à tous les agents
- Respect strict de l’architecture existante.
- Appliquer les bonnes pratiques : clean code, SOLID, DRY.
- Toujours penser sécurité : validation, permissions, auth, données sensibles.
- Ne jamais casser l’existant sans le signaler.
- Toute décision critique doit être préparée pour audit par Claude.
- Livrables attendus :
  - Code propre et fonctionnel
  - Explication technique concise
  - Liste des fichiers impactés
  - Risques connus et limites
- Interdictions :
  - Contourner la sécurité
  - Modifier le workflow sans validation
  - Prendre des décisions architecturales finales (sauf Claude)
